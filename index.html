What is K-Nearest Neighbors?

🧭 The Core Idea

KNN is a non-parametric, instance-based learner. It stores the entire training set and classifies a new point by finding its K closest neighbors, then taking a majority vote of their class labels.

There is no explicit training phase — the model is the data.

🗳️ Majority Vote

Given query point x, find the K training points nearest to x. Count votes per class. Assign the class with the most votes.

ŷ = mode{ y_i : i ∈ N_K(x) } Ties broken by smallest distance or by weighted voting (1/d).

📏 The Role of K

Small K (e.g. K=1): very jagged, low-bias, high-variance boundary. Sensitive to noise.

Large K: smoother boundary, higher bias, lower variance. May blur class boundaries.

K is the key hyperparameter — choose it via cross-validation.

⚖️ Feature Scaling Matters

KNN uses distances directly. A feature with range [0, 1000] will dominate a feature with range [0, 1] if data is not normalised.

Always standardise (z-score) or min-max scale features before applying KNN.

The Curse of Dimensionality

📦 Volume Grows Exponentially

In high dimensions, the "neighborhood" around any point becomes very thin. Most points are far from each other, making "nearest" neighbors not actually close.

Volume of unit hypercube: 1ᵈ = 1 Volume needed to cover fraction f: r = f^(1/d) d=1: r=0.10 d=10: r=0.79 d=2: r=0.32 d=100: r=0.977

💡 Practical Implications

KNN degrades as dimensions increase — distances concentrate around the same value, so all points look equally "near." This is why feature selection and dimensionality reduction (PCA) matter before applying KNN.

Rule of thumb: KNN works best with <20 features and sufficiently large datasets relative to the feature space.

Distance Metrics

📐 Euclidean (L₂)

Straight-line distance. Most common default. Assumes all dimensions equally important and isotropic.

d(p,q) = √[ Σ (pᵢ − qᵢ)² ] Sensitive to scale → always normalise!

🏙️ Manhattan (L₁)

Sum of absolute differences. Robust to outliers in individual dimensions. Preferred for high-dimensional data.

d(p,q) = Σ |pᵢ − qᵢ| Draws a diamond-shaped neighborhood instead of a circle.

🔢 Minkowski (Lₚ)

Generalises Euclidean (p=2) and Manhattan (p=1). Larger p puts more weight on the dominant dimension.

d(p,q) = [ Σ |pᵢ − qᵢ|ᵖ ]^(1/p) p=1 → Manhattan p=2 → Euclidean p→∞ → Chebyshev (max dim diff)

∞ Chebyshev (L∞)

Maximum difference across any single dimension. Forms a square neighborhood. Useful for chessboard-style movement or grid-based problems.

d(p,q) = max |pᵢ − qᵢ| Neighborhoods are squares/cubes rather than circles/spheres.

Which Metric to Use?

📊 Euclidean

Default for continuous, well-scaled features. Assumes isotropic feature space.

📦 Manhattan

Better for high-dimensional data, sparse features, or when outliers are present.

🧬 Cosine (not visualised)

For text/document data where magnitude doesn't matter, only direction (angle between vectors).

🔢 Hamming (not visualised)

For categorical or binary features — counts mismatching positions between two strings.
